How to read 先找需求拉动，再找瓶颈或利润环节；最后落到公司和数据验证。
  1. 选需求场景用上方 tabs 锁定 AI、手机、汽车工业、fab、网络光模块或 memory 主线。
  2. 读传导链顺着 timeline 看逻辑、瓶颈、领先信号和相关节点。
  3. 点产业节点在节点索引里 drill-down 到 supply/demand、周期和公司入口。
  4. 做研究动作用 cycle guide 判断是需求、库存、capex、价格还是 margin 问题。
Demand cycle先看终端需求/云capex/汽车产量/手机sell-through，它决定哪条链被拉动。
Inventory cycle模拟、手机、被动件、memory最容易被库存误导；需要区分sell-in补库和真实sell-through。
Capex cyclefoundry、memory、WFE、材料和测试更受capex、稼动率、折旧和订单/backlog影响。
Pricing / margin cyclememory看ASP，PCB/optical看ASP erosion，foundry/OSAT看utilization，ODM看pass-through和现金转换。
Industry node index点击下方任一产业节点，右侧 inspector 会同步更新。高亮节点代表当前场景链或相邻上下游。
